# Introduction

> Welcome to Goodie Forms documentation.

Welcome to [**Goodie Forms**](https://github.com/iGoodie/goodie-forms), an unopinionated modern form state and data management library.

## What is Goodie Forms?

**Goodie Forms** provides type-safe and feature-rich pieces to help you build a seamless form experience.
It is built from ground up, with a concern of DX (Developer Experience) in mind.

It is enriched and generified using these:

<card-group>
<card icon="goodie:brand-standardschema" target="\_blank" title="Standard Schema V1" to="https://standardschema.dev/">

A common interface for TypeScript validation libraries. Makes Goodie Forms validation library agnostic.

</card>

<card icon="skill-icons:typescript" target="\_blank" title="Typescript" to="https://www.typescriptlang.org/">

A strongly typed programming language that builds on JavaScript. Supercharges Goodie Forms with type-safety and IntelliSense suggestions.

</card>

<card icon="goodie:brand-immer" target="\_blank" title="Immer" to="https://immerjs.github.io/immer/">

A tiny package that allows you to work with immutable state. Makes Goodie Forms data immutable and performant to work with.

</card>

<card icon="goodie:brand-evilmartians" target="\_blank" title="Nano Events" to="https://github.com/ai/nanoevents">

Simple and tiny event emitter library for JavaScript. Gives Goodie Forms power to emit and listen to certain events.

</card>
</card-group>

## Key Features

- **Framework-agnostic core**: Designed to work with any UI layer (React, Vue, Solid, or vanilla).
- **Fully type-safe**: Built with [Typescript](https://www.typescriptlang.org/) from the ground up. Deep type inference ensures form values, errors, and validation are correctly typed at every level.
- **Validation library agnostic**: Thanks to [Standard Schema](https://standardschema.dev/), you can plug in popular libraries like [Zod](https://zod.dev/), [Valibot](https://valibot.dev/), [Yup](https://github.com/jquense/yup), or your own validation logic without being locked into a single solution.
- **Fine-grained control over state**: Track touched, dirty, validating, and error states per field or per form.
- **Optimized for DX**: Clear APIs, strong IntelliSense support, and predictable behavior reduce cognitive load during development.
- **Unopinionated by default**: No forced UI components. No design constraints. Goodie Forms handles logic — you handle presentation.
